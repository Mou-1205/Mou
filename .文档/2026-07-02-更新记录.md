## 友链测速功能修复

### 问题
友链页面测速显示 0.x ms，明显不是真实网络延迟。实际访问站点加载很慢。

### 根因
使用 `new Image()` + `img.src` 探针方案，浏览器图片缓存忽略 query string，`onload` 瞬间触发，测到的是缓存命中时间（~0ms）。

### 演进过程

| 版本 | 方案 | 问题 |
|------|------|------|
| v1（原始） | `new Image()` + `img.src` | 测到浏览器缓存，显示 ~0ms |
| v2 | `fetch` + `mode: "no-cors"` + `<1000ms` 阈值 | 变量遮蔽 + 超时不显示"未知" |
| v3（当前） | `fetch` + `mode: "no-cors"` + catch 统一返回 null | ✅ 正确区分可达/不可达 |

### 最终方案

使用 `fetch` + `mode: "no-cors"` 发起跨域 HEAD 请求：

- `mode: "no-cors"`：允许跨域请求，返回 opaque response，服务器真实响应
- `cache: "no-cache"` + 时间戳参数：强制绕过缓存
- `AbortController` 8 秒超时：防止无限等待
- catch 统一返回 null → 显示"未知"（超时/DNS 失败/网络错误）

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/pages/friends.astro` | pingUrl 函数重写（Image → fetch） |

## 全屏布局优化

### 问题
全屏状态下左右侧栏没有贴着窗口边缘，两侧留白明显。

### 根因
`PAGE_WIDTH = 90`（rem = 1440px）限制了整个布局最大宽度，所有外层容器都有 `max-w-(--page-width) mx-auto`，大屏幕两侧留白。

### 修复
`PAGE_WIDTH` 从 `90` 改为 `120`（1440px → 1920px）。

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/constants/constants.ts` | `PAGE_WIDTH` 90 → 120 |

## 侧栏 Sticky 修复

### 问题
侧栏随着页面滚动消失，不会固定在视口。只有滚到底部附近才短暂 sticky。

### 根因
1. OverlayScrollbars 在 body 内插入 `.os-viewport` 容器并设 `overflow: hidden`，破坏 `position: sticky`
2. Layout.astro 中 sidebar sticky 的 `top` 值为 `calc(1rem - var(--banner-height-extend))`，算出约 `-308px`（负值），侧栏 sticky 位置在视口上方不可见区域
3. top 区域和 sticky 区域是两个独立 div，切换时有跳动/空白

### 修复

| 改动 | 修改前 | 修改后 |
|------|--------|--------|
| OverlayScrollbars | `data-overlayscrollbars-initialize` 在 html 和 body 上 | 移除，恢复原生滚动 |
| sidebar sticky top | `calc(1rem - var(--banner-height-extend))`（≈-308px） | `2.5rem`（40px） |
| top/sticky 间距 | top 区域 `mb-4` + 无补偿 | top 区域 `mb-4` + sticky 区域保持 `gap-4` 自然衔接 |
| tags 组件 | `position: "top"` | `position: "sticky"` |

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/layouts/Layout.astro` | 移除 `data-overlayscrollbars-initialize`；sidebar sticky top 改为 `2.5rem` |
| `src/layouts/MainGridLayout.astro` | 移除临时添加的 `min-h-screen` |
| `src/components/layout/SidebarColumn.astro` | top/sticky 两段式结构保留，去掉多余间距 |
| `src/config/sidebarConfig.ts` | tags 从 `top` 改为 `sticky` |
| `src/styles/scrollbar.css` | 删除（dead code） |
| `astro.config.mjs` | 移除 `overlayscrollbars` 预打包 |
| `package.json` | 移除 `overlayscrollbars` 依赖 |

### 备注
- 右侧栏（categories、music-sidebar）已是 sticky，无需改动
- 右侧栏的 site-stats、calendar 保持 `position: "top"`（随页面滚动）
