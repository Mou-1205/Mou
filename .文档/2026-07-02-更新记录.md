## 顶栏滚动收起修复

### 问题
子页面（anime/ archive/ albums/ devices/ about/ friends/ projects/ skills/ timeline/）顶栏无法自动收起，一直显示。

### 根因
1. **Navbar.astro**：非首页直接 `return` 跳过滚动监听注册
2. **mobile-navbar.css**：移动端 CSS 选择器要求 `[data-is-home="true"]`，子页面不生效
3. **wallpaper-navbar-transparent.css**：CSS 变量切换规则要求 `body.wallpaper-transparent`，banner 模式下该 class 被移除
4. **back-to-top-handler.ts**：顶栏隐藏阈值使用 `BANNER_HEIGHT_HOME`（65%），子页面需要滚动很远才触发

### 修复
- `Navbar.astro`：移除 `!isHomePage` 强制 return 逻辑
- `mobile-navbar.css`：移除 `[data-is-home="true"]` 限制
- `wallpaper-navbar-transparent.css`：移除 `body.wallpaper-transparent` 前缀，让 CSS 变量在所有页面生效
- `back-to-top-handler.ts`：隐藏阈值从 65% 降至 20% 视口高度

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/components/organisms/navigation/Navbar.astro` | 移除非首页强制 return |
| `src/styles/mobile-navbar.css` | 移除 `[data-is-home="true"]` 限制 |
| `src/styles/wallpaper-navbar-transparent.css` | CSS 变量规则不再依赖 `body.wallpaper-transparent` |
| `src/scripts/handlers/back-to-top-handler.ts` | 隐藏阈值降至 20% |
